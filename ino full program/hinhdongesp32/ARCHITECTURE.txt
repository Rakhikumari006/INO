STATE MACHINE TRANSITIONS

Trigger Events:
• Button SHORT PRESS  → Change Emotion
• Button LONG PRESS   → Sleep Mode
• 30s Inactivity      → Sleep Mode
• Random (5-10s)      → Automatic Blink
• Sleep + Button      → Wake Up

State Cycle:
┌─────────────────────────────────────────────────────┐
│ Idle → (play animation + random blinks) → Idle      │
│  │                                          ▲        │
│  │  ┌──────────────────────────────────────┘        │
│  │  │  30 seconds no activity                        │
│  │  │                                                │
│  ▼  │                                                │
│ Sleep → (dimmed) → (any button) → Wake              │
│                                       │              │
│  (100% brightness restored) ──────────┘              │
│                                                       │
│ Short Press on Idle or Wake:                        │
│       → Select Random Emotion → Play Animation     │
│       → Return to Idle after animation             │
│                                                       │
└─────────────────────────────────────────────────────┘

BUTTON BEHAVIOR

Button Pin: D3 (GPIO0) to GND

Time < 1 second:
  └─► SHORT PRESS
      └─► Select random emotion
      └─► Enter EMOTION state
      └─► Display emotion for 4 seconds
      └─► Return to IDLE

Time > 1 second:
  └─► LONG PRESS
      └─► Enter SLEEP state
      └─► Dim display to 50%
      └─► Slower animation
      └─► Await button press to wake

Any press while sleeping:
  └─► Wake up
      └─► Play SURPRISED animation
      └─► Restore full brightness
      └─► Return to IDLE

POWER OPTIMIZATION FLOW

Operation State → Power State → Display State → CPU Usage
─────────────────────────────────────────────────────────────

IDLE/Active    → Normal      → 255 (bright)  → Full speed
               → RAM active  → Updates fast  → 42ms/frame
               
EMOTION play   → Normal      → 255 (bright)  → Full speed
               → Active      → Updates fast  → 20-45ms/frame

INTERACT       → Normal      → 255 (bright)  → Full speed
               → Active      → 3 seconds max → responsive

SLEEP          → Reduced     → 50 (dimmed)   → Slow
(after 30s)    → Power save  → Dims down     → 60ms/frame
               → Idle        → Lower current → ~20mA

WAKE           → Normal      → 255 (bright)  → Fast
               → Transition  → Fast update   → 2 seconds
               → Back to IDLE→ Full speed    → Normal ops

MEMORY LAYOUT

Flash Memory:
├─ Code (Arduino core + Libraries + Our code): ~250KB
├─ Frame Data (772 frames in PROGMEM):         ~772KB
└─ Total Flash:                                ~1.0MB/4MB

RAM Usage:
├─ Objects (animator, display, input, sm):     ~2KB
├─ Buffers (display buffer, temp vars):        ~1KB
├─ Arduino/System:                             ~5KB
├─ Used Total:                                 ~8KB/80KB
└─ Available for expansion:                    ~48-72KB

PROGMEM (Flash) Advantages:
✓ Frames don't consume RAM
✓ Allows large animation libraries
✓ Fast access from flash
✓ Survives power cycles
